## Tkinter — Interfaces Gráficas em Python

Os exercícios práticos cobriram os fundamentos da biblioteca Tkinter para construção de interfaces gráficas:

| Arquivo | Conteúdo |
|---|---|
| `01_janela_basica.py` | Criação e configuração de janela principal |
| `02_widgets.py` | Botões, labels, entradas de texto |
| `03_layout_grid.py` | Organização de elementos com sistema Grid |

| `06_grafico.py` | Integração com gráficos (matplotlib) |

## Avaliação do conteúdo

Tkinter não é minha área de interesse principal — meu foco está em níveis mais baixos da stack de software. Mas entender estruturação de layout com Grid, ciclo de vida de widgets e gestão de eventos de janela tem valor prático real: criar dashboards de monitoramento, ferramentas internas de suporte, interfaces rápidas pra sistemas que precisam de algum nível de interatividade sem depender de um frontend web.

O padrão de event-driven programming que o Tkinter usa — associar callbacks a eventos de widget — é o mesmo padrão que aparece em sistemas de UI de qualquer nível de abstração, desde bibliotecas de jogos até frameworks web. O contexto muda, a estrutura não.

Não é onde quero investir tempo profundo, mas ser autossuficiente nisso faz parte do perfil que quero construir como engenheiro — não depender de ninguém pra criar uma interface funcional quando o projeto precisar de uma.
